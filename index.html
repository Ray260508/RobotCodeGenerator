SYSTEM IDENTIFICATION GUIDELINE

Tuning with WPILib SysId

Instead of guessing PID and Feedforward values, the generated code includes a SysIdRoutine for your configured mechanisms. This allows you to mathematically characterize your robot's physical properties (kV, kA, kS) and calculate optimal PID gains.

Step 1 Bind SysId Commands

In your generated RobotContainer.java, bind the mechanism's SysId commands to your controller buttons. Make sure the robot has plenty of room to move!

// Example: Binding Elevator SysId to D-Pad
controller.povUp().whileTrue(elevator.sysIdQuasistatic(SysIdRoutine.Direction.kForward));
controller.povDown().whileTrue(elevator.sysIdQuasistatic(SysIdRoutine.Direction.kReverse));
controller.povRight().whileTrue(elevator.sysIdDynamic(SysIdRoutine.Direction.kForward));
controller.povLeft().whileTrue(elevator.sysIdDynamic(SysIdRoutine.Direction.kReverse));

Step 2 Run the Characterization Tests

Deploy the code and run the four tests in Autonomous or Teleop (depending on how you bound them). The data is automatically logged to a .wpilog file via WPILib DataLogManager or AdvantageKit.

  • Quasistatic Forward/Reverse: Slowly ramps voltage to overcome static friction (kS) and measure velocity (kV). Let it run until the mechanism reaches its limit.
  • Dynamic Forward/Reverse: Applies a sudden step voltage to measure acceleration (kA).

Step 3 Analyze with SysId Tool

Open the WPILib SysId tool from VS Code (WPILib: Start Tool -> SysId). Load your recorded .wpilog file from the roboRIO or your computer.

  1. Select the mechanism type (e.g., Simple, Elevator, Arm).
  2. Drag the logged signals (Voltage, Position, Velocity) into the analyzer UI.
  3. Adjust the window size to filter out noise, and SysId will calculate your Feedforward and PID values!

Step 4 Apply the Constants

Copy the generated kP, kD, kS, kV, kA values back into your generated Constants.java. The subsystem will automatically use these for perfectly tuned, smooth control.
